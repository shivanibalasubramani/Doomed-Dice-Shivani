-->Approach : Multiprocessing and Parallelization:

Given two six-sided dice, the task is to determine the probability of each possible sum that can result from rolling both dice. This involves finding the count of possible combinations, listing all combinations, and calculating the probability of each sum.

-->Approach : Multiprocessing and Parallelization:

1.worker Function:

This function defines the task that each process will execute. It takes a chunk of possible combinations (possibles_a_chunk) from one set of dice rolls, iterates through each combination, and checks if it matches any combination from the other set of dice rolls using the check_maps function.

2.find_matching_pair_parallel Function:

This function utilizes multiprocessing to distribute the workload across multiple processes. It divides the possible combinations of one set of dice rolls (possibles_a) into chunks and assigns each chunk to a separate process.
